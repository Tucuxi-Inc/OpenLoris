## Design System Foundation

**Design Philosophy:** Edward Tufte-inspired scientific illustration aesthetic. Think natural history field guide meets enterprise software. No neon, no glowing effects, no gradients.

> See [CLAUDE-CODE-GUIDE.md](./CLAUDE-CODE-GUIDE.md) for complete CSS specifications.

### Color Palette

| Color | Hex | Usage |
|-------|-----|-------|
| Background Primary | `#FFFEF8` | Cream/off-white - main page background |
| Background Secondary | `#FAF9F6` | Slightly darker cream - cards |
| Background Tertiary | `#F5F4F0` | Warm gray - hover states |
| Text Primary | `#1A1A1A` | Near-black - body text, headings |
| Text Secondary | `#4A4A4A` | Dark gray - secondary text |
| Text Tertiary | `#6B6B6B` | Medium gray - captions |
| Ink Accent | `#8B5A2B` | Loris brown - links, primary actions |
| Ink Success | `#2E5E4E` | Forest green - success states |
| Ink Warning | `#8B6914` | Ochre - warnings, GUD alerts |
| Ink Error | `#8B2E2E` | Burgundy - errors |
| Rule Light | `#E5E4E0` | Hairline rules |

**What NOT to use:** No `#00D4FF` (cyan), no bright neon colors, no `box-shadow` glows, no gradients.

### Typography

| Element | Font | Size | Notes |
|---------|------|------|-------|
| H1 | Georgia/Times (serif) | 36px | Tight letter-spacing |
| H2 | Georgia/Times (serif) | 30px | |
| H3 | Georgia/Times (serif) | 24px | |
| Body | Georgia/Times (serif) | 16px | 1.5 line-height for readability |
| Labels | IBM Plex Mono | 12px | ALL CAPS, wide letter-spacing |
| Code/Data | IBM Plex Mono | 14px | |

**Key principle:** Serif fonts for content (readable, bookish), monospace for labels and data (precise, functional).

## Component Specifications

> **Note:** Full CSS specifications are in [CLAUDE-CODE-GUIDE.md](./CLAUDE-CODE-GUIDE.md). Below are quick reference examples following the Tufte aesthetic.

### Primary Button

```css
/* Understated, typographic - NOT shouty */
.btn-primary {
  font-family: Georgia, 'Times New Roman', serif;
  font-size: 14px;
  font-weight: 500;
  letter-spacing: 0.02em;
  text-transform: uppercase;

  color: #FFFEF8;           /* Cream text */
  background: #1A1A1A;      /* Near-black background */

  padding: 10px 20px;
  border: 1px solid #1A1A1A;
  border-radius: 2px;       /* Subtle, NOT pill-shaped */

  transition: background 0.15s ease;
}

.btn-primary:hover {
  background: #4A4A4A;      /* Lighten slightly */
}
```

### Question Card

```css
/* Paper-like, flat, no shadow effects */
.question-card {
  background: #FAF9F6;      /* Cream surface */
  border: 1px solid #E5E4E0; /* Light hairline */
  border-radius: 2px;       /* Minimal rounding */
  padding: 24px;

  /* NO shadow - flat, printed appearance */
  box-shadow: none;
}

/* Emphasis via left rule, NOT color */
.question-card.has-new {
  border-left: 3px solid #8B5A2B;  /* Loris brown accent */
}
```

### Status Indicators

```css
/* Typographic status - NOT colorful badges */
.status {
  font-family: 'IBM Plex Mono', monospace;
  font-size: 11px;
  letter-spacing: 0.05em;
  text-transform: uppercase;
}

.status-auto-answered {
  color: #2E5E4E;           /* Forest green */
}

.status-pending {
  color: #8B6914;           /* Ochre */
}

.status-resolved {
  color: #2E5E4E;           /* Forest green */
}

.status-error {
  color: #8B2E2E;           /* Burgundy */
}

/* If background needed, keep very subtle */
.status-with-bg {
  background: #F5F4F0;      /* Warm gray */
  padding: 4px 8px;
  border-radius: 2px;       /* NOT pill-shaped */
}
```

### Hairline Rules

```css
/* Thin, understated dividers */
hr, .rule {
  border: none;
  border-top: 1px solid #E5E4E0;
  margin: 24px 0;
}

.rule-strong {
  border-top: 2px solid #1A1A1A;
}
```

## Accessibility Requirements

1. **Color contrast** - All text meets WCAG AA (4.5:1 for normal text)
2. **Keyboard navigation** - Full keyboard support for all interactions
3. **Screen readers** - Proper ARIA labels on all interactive elements
4. **Focus indicators** - Visible focus states on all focusable elements
5. **Alt text** - All Loris images have descriptive alt text
6. **Form labels** - All form inputs have associated labels
7. **Error messages** - Clear, descriptive error messages near inputs
